# 需求理解与假设说明

## 1. 关键留白点与假设

1. 分级审批金额阈值  
   物流异常 `0 <= amount < 1000` 进入一级审批，`amount >= 1000` 需要二级审批。品控异常默认直接进入二级审批。理由：物流低金额损失可由一线快速处理，高金额和来货质量问题涉及追偿/采购/库存风险，需要更高层级把关。实现为 `v3_approval_rules`，可在页面调整。

2. 审批超时时长  
   一级审批 24 小时未处理自动升级二级审批；二级审批 48 小时未处理自动驳回为 `rejected`，达到重提上限则关闭。理由：一级审批偏运营处理，应当天响应；二级审批涉及金额和责任判断，给更长处理窗口。

3. 重提次数上限  
   驳回后最多重提 2 次。理由：允许补充材料纠错，但避免工单无限循环。实现为 `max_resubmit_count` 配置。

4. 物流异常类型与下游动作映射  
   丢件：客户全额理赔 + 重新发货，扣减库存。破损：客户全额理赔 + 退货入库。客户拒收：退货入库，不默认赔付。超时未签收：按金额 10% 客户补偿。地址错误：重新发货，不默认赔付。理由：区分货损责任、履约补偿和地址纠错。

5. 角色权限划分  
   操作员可扫描、上报、重提自己的驳回工单；一级审批员只能处理一级工单；二级审批员只能处理二级工单；品控主管只能对品控工单做误判快速放行；管理员可执行维护任务和兜底操作。上报人不能审批自己提交的工单。

6. V2 数据同步频率与一致性策略  
   创建工单和扫描时实时调用 V2 校验；详情和列表展示使用 V3 快照，并标注“实时获取自 V2”或“本地缓存，同步于 XX”。接口超时 3.5 秒，重试 1 次。V2 不可用时，不允许创建新异常，但允许查看缓存，避免整体白屏。

7. 品控暂扣超时时长  
   品控暂扣 2 小时未处理强制升级二级审批，独立于审批超时。理由：暂扣会占用仓储和发货资源，压仓成本远高于普通审批等待成本，必须更短。

8. 品控规则触发阈值  
   数量差异 `>= 3%`、破损等级 `>= 2`、规格偏差 `>= 5mm`、标签不一致、批次库龄 `>= 30 天` 均触发品控异常。理由：轻微误差可由仓库现场修正，超过阈值后会影响客户收货或追偿证据链。实现为 `v3_quality_rules`，页面可调整。

9. 品控主管权限边界  
   品控主管可做“误判快速放行”，必须填写原因并写入审批记录；不默认具备一级/二级审批权限。若同一人兼任多个角色，系统仍按当前登录角色和后端权限校验执行，避免绕过审批。

## 2. 其他工程假设

- 本次实现按单租户、单仓库 `default / WH-SH-01` 建模；表结构已保留 `tenant_id`、`warehouse_id`，后续多租户需在所有查询中加租户范围过滤。
- V3 库存为自有轻量库存视图，不替代真实 WMS；真实环境可把 `v3_inventory_movements` 作为 outbox，再由任务推送 WMS。
- 本地 `/api/v2/*` 是开发适配器，用来验证 HTTP 契约；生产必须配置真实 `V2_API_BASE_URL`。
- AI 建议本次未接入主流程，保留 `ai_suggestion` 字段；若后续接入，必须仅作为人工参考，不能自动审批。

## 3. 希望向产品经理澄清的问题

- 物流异常的理赔金额是否应按商品金额、运费、订单金额还是保险金额计算？
- “客户拒收”是否需要区分客户原因、物流原因和商品质量原因？
- 品控破损等级是否已有企业标准，是否需要上传图片作为证据？
- 品控主管快速放行是否需要二次确认或抽检比例限制？
- V2 是否已有对外接口版本规范，接口字段升级是否能提供 schema 版本号？
- 异常处理结果是否必须回写 V2，以阻止 V2 继续正常发货？
- 大批量 20 万工单时，是否有 SLA、归档周期和审计保留年限要求？

## 4. 反思题

- 如果题目给出精确阈值，架构仍会保留规则表，只是默认配置改成产品值，避免后续改代码。
- 最大一致性风险在审批通过后下游联动失败；本实现用数据库事务兜底。若不用事务，应使用 outbox + 幂等消费者 + 补偿任务。
- 面对模糊需求，会先识别关键不可逆风险并提出假设，做一个可运行版本再迭代，避免长期停在讨论阶段。
- 新增“客户投诉服务态度”可作为新异常类型和动作映射接入，不需要改状态机；若无库存/赔付，只生成审批记录即可。
- 工单量到 20 万时，列表查询和超时任务全表扫描会先成为瓶颈，需要组合索引、游标分页、分片扫描和归档。
- 给 V2 新增接口时使用 `/api/v1` 版本前缀、只加字段不删字段、灰度开放；金额从 `int` 改 `decimal` 时 V3 统一按字符串/decimal 解析，避免精度丢失。
